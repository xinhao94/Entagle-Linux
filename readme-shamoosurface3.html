Compiling ENTANGLE

Introduction

The ENTANGLE source archive contains all the data, configuration files, and source code required to run ENTANGLE. The ENTANGLE source tree is written entirely in Java and can be compiled and run with the J2SE 5.0. In all likelihood, it has little dependency to the newest version of the Java environment and can be run with an earlier version of the environment.

Dependencies

ENTANGLE depends on two external libraries not included in J2SE 5.0, Java3d and Apache's Crimson XML Parser. The jar file for the Crimson project has been included in the source archive. The Java3d project includes installers for various platforms and it is assumed in the ANT build script that Java3d is installed as an extension to the JRE (Java Runtime Environment).

Simple Compilation Instructions

These compilation instructions are intended for those familiar with how to execute operating systems and compiling Java programs in general. There are two ways to compile the ENTANGLE source:

  1. Recommended: With the Ant Apache build utility installed, enter the ENTANGLE base directory with your favorite shell/command prompt. The 'src' folder should be listed in the current directory. Execute the command 'ant'. This will execute the default ant build script, "build.xml" which will compile the java sources and output the class files into the 'bin' folder.
  2. Within a command prompt/shell of your environment, enter the ENTANGLE base directory. The 'src' folder should be listed in the current directory contents. Execute the 'javac' command for every java file in the 'src' folder

Running ENTANGLE

ENTANGLE has 4 main entry points, classes with main methods. Simply execute these class files from within your favorite command prompt/shell, e.g. "java -classpath lib/crimson.jar entangle.EntryClass". These classes assume the working directory is the base ENTANGLE directory. The non-gui classes require arguments or input files to process. These non-gui classes were used by the authors of ENTANGLE to compile the protein-nucleic acid interaction databases.

  1. entangle.EntryClass - entry point to the GUI application
  2. entangle.GeneralizedBaseInteractionProcessor - command-line application which takes as input a list of PDB files, or PDB ids, (separated by new lines) and generates RIBBONS input files that display an aggregrated view of all protein-nucleic acid interactions oriented with respect to each of the nucleic acid residues
  3. entangle.PDBProcessor - command-line application which takes as input a list of PDB files, or PDB ids, (separated by new lines) and generates an XML document including all protein-nucleic acid interactions identified by the ENTANGLE program.
  4. entangle.utils.PNDatabaseMaker - command-line application which uses JDBC to create a database of interactions from XML documents generated by entangle.PDBProcessor. It is currently hardcoded to use an ODBC connection by using the url, "jdbc:odbc:Protein-RNA Interactions".
